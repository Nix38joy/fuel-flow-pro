# FuelFlow Pro

FuelFlow Pro — интерфейс управления АЗС в режиме реального времени с валидацией доменных правил, хранением состояния и модульной архитектурой.

## Что реализовано

- Управление заправкой: `start` → `pause` → `resume` → `complete` / `cancel`.
- Динамический пролив топлива (1 л/сек) с расчётом фактически налитого объёма.
- Контроль инвариантов: запрет невалидных переходов и блокировка закрытия смены при активных заправках.
- Персист состояния через `zustand/persist`.
- UI-панели: список колонок, мониторинг резервуаров, история операций и отчёт по смене.

## Архитектура

Проект построен на FSD-подходе с выделенным use-case слоем:

- `entities` — типы и модели доменных сущностей.
- `features` — пользовательские сценарии.
- `widgets` — агрегированные UI-блоки.
- `app/use-cases` — чистые доменные операции и правила переходов.
- `app/providers/store.ts` — Zustand store как orchestration-слой между UI и use-cases.

### Доменный контракт

Операции use-case слоя возвращают единый `ActionResult`:

- `{ ok: true, data }` для успешного перехода.
- `{ ok: false, error }` для отказа по бизнес-правилу.

Это позволяет хранить инварианты в одном месте и не размазывать их по компонентам.
